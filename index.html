NSOperation理解

发表于 2017-05-03   |   分类于 iOS   |  

主要说一下Asynchronous和Synchronous的区别，Concurrent和Non-concurrent的区别。

Asynchronous Versus Synchronous Operations

Operation默认是synchronous，synchronous的operation执行时不需要创建额外的线程，只是在当前线程执行，task执行完成后返回。
Asynchronous的operation在start后不等待task执行完就立即返回，其task会在另一个线程中执行。

根据文档的描述，放在NSoperationQueue中的operation都是asynchronous的。

dispatch_sync/async与并发串行队列的理解

发表于 2017-05-02   |   分类于 iOS   |  

主要针对下面这个例子出现的各种情况来分析iOS的GCD。

The following code to call [self goToNext] from main thread have different result with different dispatch_xxxx for <1> and <2>.

  1. dispatch_sync & dispatch_sync, the result is deadlock;
  2. dispatch_async & dispatch_async, the result is NULL;
  3. dispatch_sync & dispatch_async, the result is deadlock;
  4. dispatch_async & dispatch_sync, the result is NULL.

在一个串行队列中，在第一种情况下，在队列中插入了blockA，然后blockA中又在队列中插入了blockB。在串行队列中，需要先执行blockA，再执行blockB；而在blockA中需要同步等待blockB完成。所以就行成了blockA等待blockB，blockB等待blockA的死锁局面。

与上面情况相同，如果这个队列是并行队列的话，这种互相等待的情况就不会发生。

UITableView在iOS6的变化

发表于 2017-04-20   |   分类于 iOS   |  

dequeueReusableCellWithIdentifier:forIndexPath和dequeueReusableCellWithIdentifier:的区别
1> IndexPath版本没有注册identifier的情况下，会crash；旧版本会返回nil。所以IndexPath版本必须注册。
2> IndexPath永远会得到一个initialized的cell；旧版本会在没有可重用的cell时返回nil。所以IndexPath版本无需写cell为空判断[https://developer.apple.com/videos/wwdc/2012/?id=200]，8分30秒左右，旧版本必须写cell为空判断。

iOS6以上选择使用IndexPath版本。

ReactiveCocoa基础概念理解

发表于 2017-03-02   |   分类于 ReactiveCocoa   |  

基于一个基本的RACSignal使用方式:

signal & didSubscribe

创建signal时需要提供signal需要处理的一些事情(这些事情是由某一个调度去异步处理的)，需要处理的事情是一个叫做didSubscribe的block，这个didSubscribe最终保存在signal的属性_didSubscribe中。

【当signal被subscribeNext了，就会执行signal的didSubscribe】

subscribe & subscriber & subscribeNext

当对Signal执行subscribeNext时，提供了一个nextBlock，此时会创建一个RACSubscriber，RACSubscriber的next成员保存了nextBlock，然后在去调用subscribe。
subscribe去调度signal的didSubscribe，而didSubscribe这个block的参数就是发起subscribeNext的RACSubscriber。

sendNext

在异步调度didSubscribe的过程中，会对Signal的subscribeNext发起者RACSubscriber调用sendNext，执行RACSubscriber的nexBlock。

replayLazily和replay的区别

replay里自动connect了，自动订阅了sourceSignal；replayLazily需要用户自己来订阅，而订阅的内容只执行一次。
